# SeismicPlanet

An interactive 3D time-series map visualizing earthquakes worldwide. Stream or replay historical seismic data over a 3D globe, explore earthquake history by time window, and filter by magnitude and depth.

Data sourced from the [USGS Earthquake Hazards Program](https://earthquake.usgs.gov/fdsnws/event/1/).

---

## Requirements

- Python 3.11+
- Node.js 18+

---

## Setup

### 1. Data pipeline

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

### 2. Frontend

```bash
cd web
npm install
```

---

## Getting data

Run a full sync to download historical earthquake records and export them for the web:

```bash
sp-sync
```

This downloads all years from 1900 to present (skipping years already on disk), compacts daily files into yearly archives, and exports web-ready JSON to `web/public/data/`.

Safe to re-run — existing yearly files are not re-downloaded.

**First run takes a while.** Subsequent runs are fast (only missing days are fetched).

### Optional: globe texture

Download a 4096×2048 equirectangular Earth texture (NASA Blue Marble or similar) and place it at:

```
web/public/textures/earth.jpg
```

Without it the globe renders as a faint blue tint.

## Pipeline commands

| Command | Purpose |
|---|---|
| `sp-sync` | **Recommended.** Full incremental sync — download, compact, and export |
| `sp-download` | Bulk download historical events into `data/YYYY.parquet` |
| `sp-append` | Fetch yesterday's events into `data/daily/YYYY-MM-DD.parquet` |
| `sp-compact` | Merge daily files into the yearly archive (deduplicates by event ID) |
| `sp-today` | Live fetch from midnight UTC to now; outputs GeoJSON |
| `sp-export` | Export Parquet archive to web-ready JSON chunks |
| `sp-query "<SQL>"` | Ad-hoc DuckDB query across all Parquet files |

### Example queries

```bash
sp-query "SELECT COUNT(*) FROM events"
sp-query "SELECT * FROM events WHERE magnitude >= 8.0 ORDER BY magnitude DESC LIMIT 20"
sp-query "SELECT year(epoch_ms(time)) yr, COUNT(*) n FROM events GROUP BY yr ORDER BY yr"
```

### Daily cron

```cron
0 1 * * *  cd /path/to/seismicplanet && .venv/bin/sp-sync
```

## Project structure

```
seismicplanet/
├── pipeline/          # Python data pipeline
│   ├── sync.py        # sp-sync: full incremental sync
│   ├── download.py    # sp-download: bulk historical fetch
│   ├── export.py      # sp-export: Parquet → web JSON
│   ├── today.py       # sp-today: live current-day fetch
│   └── schema.py      # Shared Parquet schema
├── data/              # Parquet archive (gitignored)
│   ├── 1900.parquet
│   ├── ...
│   └── daily/
└── web/               # React + Three.js frontend
    ├── public/
    │   ├── data/      # Exported JSON (generated by sp-sync)
    │   └── textures/  # Optional earth texture
    └── src/
        ├── components/    # 3D scene: globe, earthquake points, borders
        ├── hooks/         # Data loading and caching
        ├── ui/            # Timeline, panels, controls
        └── store.ts       # Zustand state (playback, filters)
```
